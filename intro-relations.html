    Section 5.1 Introduction to Relations

    A relation \(R\) from set \(A\) into set \(B\) is defined as a subset of the Cartesian product \(A \times B\text{,}\) represented as:
    \begin{equation*} R \subseteq A \times B \end{equation*}
    The Cartesian product \(A \times B\) consists of all possible ordered pairs \((a, b)\text{,}\) where \(a \in A\) and \(b \in B\text{.}\) Each pair combines an element from set \(A\) with an element from set \(B\text{.}\)
    Let’s define two sets, Pants and Shirts, as examples:
    \begin{equation*} \text{Pants} = \{p_1, p_2, p_3, p_4\} \\ \end{equation*}
    \begin{equation*} \text{Shirts} = \{s_1, s_2, s_3, s_4\} \end{equation*}
    The Cartesian product of Pants and Shirts includes all possible combinations of pants with shirts.
    To explore how we can match pants with shirts based on their style, we define a relation as a subset of this Cartesian product, where each pair matches by a certain criterion, such as style.
    First, let’s define a set of Styles, and then create relations for Pants and Shirts based on these styles.
    \begin{equation*} \text{Styles} = \{\text{athletic}, \text{formal}\} \end{equation*}
    \begin{equation*} \text{PantStyles} = \{(p_1, \text{athletic}), (p_2, \text{formal}), (p_3, \text{athletic}), (p_4, \text{formal})\} \end{equation*}
    \begin{equation*} \text{ShirtStyles} = \{(s_1, \text{athletic}), (s_2, \text{formal}), (s_3, \text{athletic}), (s_4, \text{formal})\} \end{equation*}
    We can visualize all possible combinations of pants and shirts, taking their styles into account.
    With these combinations, let’s define a specific relation from Pants to Shirts based on matching styles. This relation is a subset of the Cartesian product PantStyles \(\times\) ShirtStyles, including only those pairs where the pants and shirts have the same style. This is a second example of a relation, on another relation.
    \begin{equation*} \text{MatchingOutfit} \subseteq \text{PantStyles} \times \text{ShirtStyles} \end{equation*}
